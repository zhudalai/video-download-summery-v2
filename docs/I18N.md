## 1. 语言代码约定

### 1.1 BCP-47 标准

所有语言代码遵循 [BCP 47](https://tools.ietf.org/html/bcp47) 标准(语言-区域),避免使用 `zh-CN` 与 `zh_TW` 等非标准形式。

| 语言 | BCP-47 代码 | V 版本 | Stripe locale | 状态 |
|------|------------|--------|---------------|------|
| 英语(English) | `en` | V1 | `en` | 已上线 |
| 中文简体(Simplified Chinese) | `zh-CN` | V1 | `zh` | 已上线 |
| 日语(Japanese) | `ja` | V1 | `ja` | 已上线 |
| 韩语(Korean) | `ko` | V2 | `ko` | 规划中 |
| 西班牙语(Spanish) | `es` | V2 | `es` | 规划中 |
| 繁体中文(Traditional Chinese) | `zh-TW` | V2 | `zh-TW` | 规划中 |
| 德语(German) | `de` | V3(预留) | `de` | 未开始 |
| 法语(French) | `fr` | V3(预留) | `fr` | 未开始 |
| 葡萄牙语(巴西) | `pt-BR` | V3(预留) | `pt-BR` | 未开始 |

### 1.2 语言代码与 Stripe locale 映射表

Stripe 支持 30+ 种语言的自动翻译,但代码与 BCP-47 不完全一致(例如 Stripe 用 `zh` 代表简体中文)。创建显式映射避免隐式转换出错:

```typescript
// src/i18n/stripe-locale.ts

export const STRIPE_LOCALE_MAP: Record<AppLocale, StripeLocale> = {
  'en':   'en',
  'zh-CN': 'zh',
  'ja':   'ja',
  'ko':   'ko',
  'es':   'es',
  'zh-TW': 'zh-TW',
  // V3 预留
  'de':   'de',
  'fr':   'fr',
  'pt-BR': 'pt-BR',
};

export type AppLocale = keyof typeof STRIPE_LOCALE_MAP;

export function toStripeLocale(appLocale: AppLocale): StripeLocale {
  return STRIPE_LOCALE_MAP[appLocale] ?? 'en';
}
```

```python
# app/i18n/stripe_locale.py

from typing import Literal

AppLocale = Literal[
    "en", "zh-CN", "ja", "ko", "es", "zh-TW",
    "de", "fr", "pt-BR",  # V3 预留
]

STRIPE_LOCALE_MAP: dict[AppLocale, str] = {
    "en": "en",
    "zh-CN": "zh",
    "ja": "ja",
    "ko": "ko",
    "es": "es",
    "zh-TW": "zh-TW",
    "de": "de",
    "fr": "fr",
    "pt-BR": "pt-BR",
}

def to_stripe_locale(app_locale: AppLocale) -> str:
    return STRIPE_LOCALE_MAP.get(app_locale, "en")
```

### 1.3 未来扩展原则

- 新增语言只需:① 在 `AppLocale` 联合类型中加一项;② 在 `locales/` 下增加对应 JSON;③ 在 `STRIPE_LOCALE_MAP` 补映射。
- 不假设所有语言都已上线——通过 `SUPPORTED_LOCALES` 运行时白名单控制。
- 所有"语言代码"在代码层面一律使用 BCP-47,仅在调用 Stripe / 第三方 API 时转换为对方格式。

```typescript
// src/i18n/config.ts
export const SUPPORTED_LOCALES: AppLocale[] = ['en', 'zh-CN', 'ja'];
export const DEFAULT_LOCALE: AppLocale = 'en';
export const FALLBACK_LOCALE: AppLocale = 'en';
```

## 2. 前端 i18n

### 2.1 vue-i18n 配置

使用 Vue 3 Composition API 模式(`legacy: false`),配合 `@intlify/unplugin-vue-i18n` 在编译期做模板类型检查。

```typescript
// src/i18n/index.ts

import { createI18n } from 'vue-i18n';
import en from './locales/en/common.json';
import zhCN from './locales/zh-CN/common.json';
import ja from './locales/ja/common.json';

export const i18n = createI18n({
  legacy: false,                       // 必须:Composition API 模式
  locale: 'en',                        // 默认,会被 useLocale() 覆盖
  fallbackLocale: 'en',                // 缺失 key 时 fallback
  missingWarn: import.meta.env.DEV,    // 开发期警告
  fallbackWarn: import.meta.env.DEV,
  messages: {
    en,
    'zh-CN': zhCN,
    ja,
  },
});

export function setLocale(locale: AppLocale) {
  i18n.global.locale.value = locale;
  document.documentElement.setAttribute('lang', locale);
}
```

### 2.4 懒加载 + TypeScript 类型校验

#### 2.4.1 懒加载语言包

初始只加载当前语言,切换时动态 import:

```typescript
// src/i18n/lazy-load.ts

const localeCache = new Map<string, Promise<any>>();

export async function loadLocaleMessages(locale: AppLocale) {
  if (localeCache.has(locale)) return localeCache.get(locale);

  const promise = import(`./locales/${locale}/common.json`)
    .then(m => m.default);
  localeCache.set(locale, promise);
  return promise;
}
```

#### 2.4.2 TypeScript 类型校验

使用 `vue-tsc` 编译期检查缺失 key。通过 `unplugin-vue-i18n` 生成类型定义:

```json
// tsconfig.json
{
  "compilerOptions": {
    "types": ["@intlify/unplugin-vue-i18n/globals"]
  }
}
```

```typescript
// src/i18n/typed-keys.ts
import type en from './locales/en/common.json';

type NestedKeys<T, Prefix extends string = ''> = T extends object
  ? { [K in keyof T & string]: NestedKeys<T[K], Prefix extends '' ? K : `${Prefix}.${K}`> }[keyof T & string]
  : Prefix;

export type TranslationKey = NestedKeys<typeof en>;
```

```bash

vue-tsc --noEmit
```

任何缺失 key 在 `vue-tsc` 严格模式下会报错,阻止构建。

### 2.5 语言切换 7 步 Checklist

切换语言时严格按顺序执行以下步骤,任何一步失败需回滚:

```typescript
// src/composables/useLocale.ts

import { setLocale } from '@/i18n';
import { loadLocaleMessages } from '@/i18n/lazy-load';
import { useUserStore } from '@/stores/user';
import { useSeo } from '@/composables/useSeo';
import { useAiSummary } from '@/composables/useAiSummary';
import { apiClient } from '@/api/client';

export async function switchLocale(newLocale: AppLocale): Promise<void> {
  const currentLocale = i18n.global.locale.value;
  if (currentLocale === newLocale) return;

  // Step 1: 立即更新 <html lang> 属性(避免屏幕阅读器读错)
  document.documentElement.setAttribute('lang', newLocale);

  try {
    // Step 2: 懒加载目标语言包
    const messages = await loadLocaleMessages(newLocale);
    i18n.global.setLocaleMessage(newLocale, messages);

    // Step 3: 切换 vue-i18n 当前 locale
    setLocale(newLocale);

    // Step 4: PATCH /me/language 持久化到后端
    const userStore = useUserStore();
    await userStore.updateLanguage(newLocale);

    // Step 5: 刷新当前页翻译(路由组件自动响应,无需手动)

    // Step 6: 重请求 API(让后端错误消息等返回新语言)
    apiClient.defaults.headers.common['X-Language'] = newLocale;
    // 触发当前页数据重新加载
    await useCurrentRoute().reload();

    // Step 7: 刷新 OG 标签 + AI 缓存
    useSeo().updateOgLocale(newLocale);
    useAiSummary().invalidateOnLocaleChange(currentLocale, newLocale);

  } catch (err) {
    // 回滚
    document.documentElement.setAttribute('lang', currentLocale);
    setLocale(currentLocale);
    throw err;
  }
}
```

**7 步 Checklist 表:**

| 步骤 | 操作 | 失败影响 | 是否可回滚 |
|------|------|----------|-----------|
| 1 | 更新 `<html lang>` | 屏幕阅读器误读 | 是 |
| 2 | 加载语言包 | 翻译缺失 | 是 |
| 3 | 切换 vue-i18n locale | 翻译不生效 | 是 |
| 4 | PATCH /me/language | 下次登录仍是旧语言 | 是 |
| 5 | 刷新翻译 | 翻译不生效 | 是 |
| 6 | 重请求 API | 后端错误仍是旧语言 | 是 |
| 7 | 刷新 OG + AI 缓存 | SEO/AI 结果旧语言 | 是 |

## 3. 后端 i18n

### 3.1 依赖注入 get_locale()

```python
# app/deps/locale.py

from fastapi import Request, Header, Depends
from typing import Optional

DEFAULT_LOCALE = "en"
SUPPORTED_LOCALES = {"en", "zh-CN", "ja", "ko", "es", "zh-TW"}

async def get_locale(
    request: Request,
    x_language: Optional[str] = Header(None, alias="X-Language"),
    current_user=Depends(get_current_user_optional),
) -> str:
    """解析优先级: current_user.language > X-Language header > Accept-Language > en"""

    # 1. 已认证用户:从 DB 取偏好
    if current_user and current_user.language:
        if current_user.language in SUPPORTED_LOCALES:
            return current_user.language

    # 2. 前端显式 header
    if x_language and x_language in SUPPORTED_LOCALES:
        return x_language

    # 3. Accept-Language header 作为后备
    accept_lang = request.headers.get("accept-language", "")
    if accept_lang:
        # 简单取第一个,生产可用 langneg 库做更精细协商
        primary = accept_lang.split(",")[0].split("-")[0].strip()
        if primary in SUPPORTED_LOCALES:
            return primary

    return DEFAULT_LOCALE


# 注册为依赖
LocaleDep = Depends(get_locale)
```

### 3.2 LocalizedHTTPException 基类

```python
# app/exceptions/i18n.py

from fastapi import HTTPException
from app.i18n import get_message  # 见 3.3

class LocalizedHTTPException(HTTPException):
    """所有业务异常继承此类,自动根据 locale 翻译 message"""

    def __init__(
        self,
        status_code: int,
        message_key: str,
        locale: str = "en",
        **kwargs,  # 插值参数 {name} {price} 等
    ):
        # 从翻译文件取文案,缺失则 fallback 到 key 本身
        translated = get_message(message_key, locale, **kwargs)
        super().__init__(status_code=status_code, detail=translated)
        self.message_key = message_key  # 前端可据此做二次处理


class PaymentRequiredError(LocalizedHTTPException):
    def __init__(self, locale: str = "en"):
        super().__init__(402, "error.payment_required", locale=locale)


class VideoNotFoundError(LocalizedHTTPException):
    def __init__(self, video_id: str, locale: str = "en"):
        super().__init__(404, "error.video_not_found", locale=locale, video_id=video_id)
```

```python
# app/i18n/loader.py

import json
from pathlib import Path
from functools import lru_cache

LOCALES_DIR = Path(__file__).parent / "messages"  # errors.{lang}.json 所在目录

@lru_cache(maxsize=None)
def load_messages(locale: str) -> dict[str, str]:
    path = LOCALES_DIR / f"errors.{locale}.json"
    if not path.exists():
        # fallback 到英语
        path = LOCALES_DIR / "errors.en.json"
    return json.loads(path.read_text(encoding="utf-8"))


def get_message(key: str, locale: str = "en", **kwargs) -> str:
    messages = load_messages(locale)
    template = messages.get(key) or load_messages("en").get(key, key)
    return template.format(**kwargs) if kwargs else template
```

### 3.4 Exception Handler 注册

```python
# app/main.py

from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
from app.exceptions.i18n import LocalizedHTTPException
from app.deps.locale import get_locale

app = FastAPI()

@app.exception_handler(LocalizedHTTPException)
async def localized_exception_handler(request: Request, exc: LocalizedHTTPException):
    """所有 LocalizedHTTPException 自动翻译后返回"""
    locale = await get_locale(request)
    # 已在构造时翻译,直接返回
    return JSONResponse(
        status_code=exc.status_code,
        content={
            "error": {
                "code": exc.message_key,
                "message": exc.detail,
            }
        },
    )
```

**Accept-Language header 作为后备:**

前端显式传 `X-Language` 优先级最高;未传时后端会读 `Accept-Language`。这保证了:
- 浏览器首次访问(未登录、未切换语言)也能看到对应语言。
- 爬虫(如 Googlebot)的 `Accept-Language` 不影响 SEO(SEO 靠 URL 子目录)。

### 4.2 缓存键设计

```python
# app/services/ai/cache.py

import hashlib

def build_summary_cache_key(video_id: str, language: str, prompt_version: str = "v1") -> str:
    """缓存键包含 language,切换语言必然 miss"""
    raw = f"{video_id}:{language}:{prompt_version}"
    hashed = hashlib.sha256(raw.encode()).hexdigest()[:16]
    return f"summary:{hashed}"
```

**缓存键组成:**
- `video_id`: 视频唯一标识
- `language`: BCP-47 语言代码
- `prompt_version`: prompt 版本号(更新 prompt 时所有缓存失效)

### 4.3 语言切换强制重新生成

```typescript
// src/composables/useAiSummary.ts

import { ref, watch } from 'vue';
import { useLocale } from '@/composables/useLocale';

export function useAiSummary() {
  const { locale } = useLocale();
  const summary = ref('');
  const isRegenerating = ref(false);
  const error = ref('');

  async function generate(videoId: string) {
    isRegenerating.value = true;
    try {
      const res = await api.post(`/videos/${videoId}/summary`, null, {
        headers: { 'X-Language': locale.value },
      });
      summary.value = res.data.summary;
    } catch (e) {
      error.value = e.response?.data?.error?.message ?? 'Unknown error';
    } finally {
      isRegenerating.value = false;
    }
  }

  function invalidateOnLocaleChange(from: string, to: string) {
    // 语言切换:清空当前总结,UI 显示"检测到语言切换,正在重新总结..."
    summary.value = '';
    // 调用方负责重新触发 generate()
  }

  return { summary, isRegenerating, error, generate, invalidateOnLocaleChange };
}
```

## 5. SEO 多语言

### 5.1 URL 子目录结构

```
https://example.com/en/download      # 英语
https://example.com/zh-CN/download   # 中文简体
https://example.com/ja/download      # 日语
https://example.com/ko/download      # 韩语 (V2)
https://example.com/es/download      # 西班牙语 (V2)
https://example.com/zh-TW/download   # 繁体中文 (V2)
```

**规则:**
- 默认语言 `en` 不重定向到 `/en/`,而是直接在根目录(`/en/` 301 到 `/` 可选,但保留显式前缀方便调试)。
- 所有语言子目录结构完全一致。
- 切换语言时保留路径: `/en/download` → `/zh-CN/download`。

```typescript
// src/composables/useLocale.ts (续)

export function switchLocalePath(targetLocale: AppLocale): string {
  const route = useRoute();
  const path = route.fullPath; // e.g. /en/download?tab=url
  const segments = path.split('/');
  segments[1] = targetLocale;  // 替换语言段
  return segments.join('/');
}
```

### 5.2 hreflang 标签

使用 `@unhead/vue` 的 `useHead` 在每个页面注入:

```typescript
// src/composables/useSeo.ts

import { useHead } from '@unhead/vue';
import { computed } from 'vue';
import { SUPPORTED_LOCALES, DEFAULT_LOCALE } from '@/i18n/config';

export function useSeo() {
  const { locale } = useLocale();
  const route = useRoute();

  const hreflangLinks = computed(() => {
    const base = 'https://example.com';
    const path = route.fullPath.replace(/^\/(en|zh-CN|ja|ko|es|zh-TW)/, '');

    const links = SUPPORTED_LOCALES.map(lang => ({
      rel: 'alternate',
      hreflang: lang,
      href: `${base}/${lang}${path}`,
    }));

    // x-default:指向默认语言(搜索引擎入口)
    links.push({
      rel: 'alternate',

}
```

## 8. 翻译管理流程

### 8.1 混合模式

```
┌─────────────────────────────────────────────────────────────┐
│                    翻译生命周期                              │
├─────────────┬───────────────────┬───────────────────────────┤
│  新 key 加入 │  机器翻译初始化     │  人工审校关键板块          │
│  (开发者)    │  (DeepL/谷歌翻译)  │  (母语编辑)               │
└─────────────┴───────────────────┴───────────────────────────┘
```

**流程:**

1. 开发者新增 key → 默认填英语 placeholder
2. CI 检测到缺失翻译 → 调用机器翻译 API 自动补全
3. 翻译管理平台(phrase.com / Crowdin)通知母语编辑审校
4. 关键板块(支付、按钮、错误、SEO)必须 100% 人工
5. 非关键板块允许机器翻译 + 人工抽校
6. 上线当日日语必须 100% 完成(合同要求)

### 8.2 翻译管理平台对比

| 平台 | 优势 | 劣势 | 推荐场景 |
|------|------|------|----------|
| phrase.com | 与 GitHub 集成好,支持 ICU MessageFormat | 价格较高 | 中大型项目 |
| Crowdin | 开源项目免费,社区审校强 | 集成略复杂 | 开源/社区驱动 |
| GitLocalize | GitHub Action 驱动,MR 工作流 | 功能较少 | 小型项目 |

**推荐:** phrase.com(与 GitHub 深度集成,支持 key 上下文截图)。

### 8.3 关键板块必须 100% 人工

以下板块不允许机器翻译直接上线:

- 支付相关文案(价格、条款、退款说明)
- 按钮文案(下载、订阅、取消)
- 错误提示(表单校验、API 错误)
- SEO meta(title, description, og:title)
- 邮件模板(欢迎、支付、DMCA)
- 法律条款(退款政策、订阅条款)

### 8.4 翻译覆盖率 ≥ 98% 才上线

```python
# scripts/check_i18n_coverage.py

import json
from pathlib import Path

SOURCE = "en"  # 源语言
THRESHOLD = 0.98

def compute_coverage(locale: str) -> float:
    src = json.loads(Path(f"src/locales/{SOURCE}/common.json").read_text())
    tgt = json.loads(Path(f"src/locales/{locale}/common.json").read_text())

    total = len(src)
    translated = sum(1 for k in src if k in tgt and tgt[k] != "")
    return translated / total

if __name__ == "__main__":
    for locale in ["zh-CN", "ja", "ko", "es", "zh-TW"]:
        coverage = compute_coverage(locale)
        print(f"{locale}: {coverage:.1%}")
        if coverage < THRESHOLD:
            raise SystemExit(f"{locale} below {THRESHOLD:.0%}")
```

## 9. 翻译覆盖率监控

### 9.1 按语言统计缺失 key 数量

```python
# scripts/i18n_report.py

import json
from pathlib import Path
from collections import defaultdict

LOCALES_DIR = Path("src/locales")
SOURCE = "en"

def get_all_keys(prefix: str, obj: dict) -> list[str]:
    keys = []
    for k, v in obj.items():
        full = f"{prefix}.{k}" if prefix else k
        if isinstance(v, dict):
            keys.extend(get_all_keys(full, v))
        else:
            keys.append(full)
    return keys

def generate_report() -> dict:
    src_file = LOCALES_DIR / SOURCE / "common.json"
    src = json.loads(src_file.read_text())
    src_keys = set(get_all_keys("", src))

    report = {}
    for locale_dir in LOCALES_DIR.iterdir():
        if not locale_dir.is_dir() or locale_dir.name == SOURCE:
            continue
        tgt_file = locale_dir / "common.json"
        if not tgt_file.exists():
            report[locale_dir.name] = {"missing": len(src_keys), "total": len(src_keys)}
            continue
        tgt = json.loads(tgt_file.read_text())
        tgt_keys = set(get_all_keys("", tgt))
        missing = src_keys - tgt_keys
        report[locale_dir.name] = {
            "missing": len(missing),
            "total": len(src_keys),
            "coverage": f"{(len(src_keys) - len(missing)) / len(src_keys):.1%}",
            "missing_keys": sorted(missing),
        }
    return report

if __name__ == "__main__":
    import pprint
    pprint.pprint(generate_report())
```

### 9.2 CI 校验前后端 key 一致性

```python
# scripts/check_i18n_parity.py

import json
from pathlib import Path

FRONTEND_ERRORS = Path("src/locales/en/errors.json")
BACKEND_ERRORS = Path("app/i18n/errors.en.json")

def check_parity():
    fe = json.loads(FRONTEND_ERRORS.read_text())
    be = json.loads(BACKEND_ERRORS.read_text())

    fe_keys = set(fe.keys())
    be_keys = set(be.keys())

    only_fe = fe_keys - be_keys
    only_be = be_keys - fe_keys

    if only_fe:
        print(f"❌ Keys only in frontend: {only_fe}")
    if only_be:
        print(f"❌ Keys only in backend: {only_be}")
    if only_fe or only_be:
        raise SystemExit(1)
    print("✅ Frontend and backend error keys are in parity")

if __name__ == "__main__":
    check_parity()
```

```typescript
export function formatDate(date: Date, locale: AppLocale): string {
  return new Intl.DateTimeFormat(locale, {
    year: 'numeric',
    month: 'long',
    day: 'numeric',
  }).format(date);
}

export function formatCurrency(amount: number, locale: AppLocale, currency = 'USD'): string {
  return new Intl.NumberFormat(locale, {
    style: 'currency',
    currency,
  }).format(amount);
}

export function formatNumber(n: number, locale: AppLocale): string {
  return new Intl.NumberFormat(locale).format(n);
}

// 示例:
// formatCurrency(9.99, 'en')      → "$9.99"
// formatCurrency(9.99, 'zh-CN')   → "US$9.99"
// formatCurrency(9.99, 'ja')      → "9.99 USD"
```

```python
# app/utils/format.py

from babel import numbers, dates
from babel.core import Locale

def format_currency(amount: float, locale: str, currency: str = "USD") -> str:
```
